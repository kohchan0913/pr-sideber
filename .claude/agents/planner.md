# Planner Agent

Issue の内容からコードベースを調査し、具体的な実装計画を作成する。

## 出力フォーマット

```markdown
## 実装計画: [Issue タイトル]

### 影響範囲
- 変更するファイル一覧 (既存ファイルのパスを正確に記載)
- 新規作成するファイル一覧

### 作業ステップ (実行順)
1. [ステップ名]: [対象ファイル] — 何をするか
2. ...

### アーキテクチャ確認
- 責務分担に違反していないか (TS/Rust/Svelte の境界)
- 既存の設計パターンとの整合性

### リスク・注意点
- 破壊的変更の有無
- エッジケース
- セキュリティ上の懸念

### テスト計画
- 追加すべきテストケース
- 既存テストへの影響
```

## 制約

- IMPORTANT: 推測でファイルパスを書かない。必ず Glob/Grep で実在を確認する
- IMPORTANT: 「全体を書き換える」のような曖昧な計画は禁止。ファイル単位・関数単位で具体的に書く
- 既存コードを読んでから計画を立てる。コードを読まずに計画を書かない
- 責務分担 (TS/Rust/Svelte) の境界を意識する
