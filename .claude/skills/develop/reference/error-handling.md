# 異常時の対応

ワークフロー中に問題が発生したらこのファイルを参照する。

## 判断基準と対応

| 状態 | 対応 |
|------|------|
| planner が「Issue が曖昧で計画を立てられない」 | ワークフロー停止。ユーザーに Issue の詳細化を依頼 |
| planner に2回差し戻しても計画が改善しない | ワークフロー停止。ユーザーに報告 |
| implementer が「計画外の設計変更が必要」と報告 | ワークフロー停止。ユーザーに判断を仰ぐ |
| レビューで CRITICAL 指摘 | 即座にユーザーに報告。自動修正しない |
| 修正ループ3回到達 | ワークフロー停止。指摘内容と修正履歴をユーザーに報告 |
| テスト不通過が続く | 原因分析結果をユーザーに報告して判断を仰ぐ |
| agent がサイレントフォールバックしている疑い | 実装結果の catch ブロックと `|| true` を grep で確認 |

## 報告フォーマット

```
## ワークフロー停止報告

**Issue:** #番号
**停止フェーズ:** Phase X (名前)
**停止理由:** (具体的に)
**これまでの作業:** (完了した Phase のサマリー)
**判断が必要な点:** (ユーザーに何を決めてほしいか)
```
